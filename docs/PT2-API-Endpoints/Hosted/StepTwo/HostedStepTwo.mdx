## Configuring the PayTabs API Endpoints

1. The first thing you will need to provide the proper endpoint URL to your request, as shown below:

    ```curl
    curl --location --request POST 'https://secure.PayTabs.com/payment/request' \
    ```
    :::caution
    <br/>
    {urls.region}
    Please note that not using the proper endpoint URI \{\{domain\}\} will lead to authentication issues within your responses. To find the your proper domain you can read our [What Is My (Region)/(endpoint URL)? solution article]({urls.region}).

2. Then you will need to provide your API server key.
    ```curl
    
    curl --location --request POST 'https://secure.PayTabs.com/payment/request' \--header 'authorization: SJ******MD-J2******L6-99******DZ' \

    ```   
    :::info
    To know how to find your API server key we recommend you to check our [How to get my Authentication/integration/API Keys?]({urls.findkeys}) solution article
    ::: 

3.  Finally, you will need to provide your profile_id within the required parameters that should be sent in the request payload.
    ```json
    {profile_id => 2342,}
    ```  
    :::info
    To know how to find your profile_id we recommend you check our [How to get your account information from PT2 dashboard?]({urls.findmid}) solution article 
    ::: 

    :::info
    In order to understand the types of PayTabs merchant profiles, we highly reccomend you to check our  [What is test profile vs live profile?]({urls.testvslive}) solution article
    ::: 


Here you are, done with configuring the PayTabs API endpoints. And now you can go on the ground with your business using the preferred payment methods.
